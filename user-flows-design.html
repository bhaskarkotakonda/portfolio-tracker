Design Risks, Options, and Loopholes

Loophole: file-level dedup only. Same holdings exported on a new date produce new hashes and repeated snapshots. Option: add a normalized holdings hash (ticker+shares+cost) to detect semantic duplicates.
Loophole: weak provenance scoring. Tweets can become “ground truth” too easily. Option: add evidence tiers (company filing > earnings call > reputable source > social) and weight theme updates by tier.
Loophole: stale prices in decisions. Daily close may mislead after-hours or pre-market recommendations. Option: include freshness metadata and confidence downgrade when price age exceeds threshold.
Tradeoff: markdown-only simplicity vs query power. Current design is excellent for auditability, but cross-cut metrics (lot aging, AIRA backtests) will get expensive. Option: keep markdown as source of truth and add a derived SQLite index later.
Tradeoff: LLM consistency. Prompt-only extraction can drift over time. Option: enforce a strict JSON schema and validator before writing frontmatter.
Strength: bounded auto-sections. Clear marker-based writes protect analyst notes and avoid accidental overwrite.
Strength: account-aware rules. Fidelity-first, RH tax warnings, MS no-trade constraint are explicit and operationally useful.
