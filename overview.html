Finance Tracker

A local-only personal finance system — consolidates 3 brokerages, enriches with LLM research, delivers intelligence via Telegram.

🏦 3 Brokerages

Robinhood (taxable), Fidelity (Roth IRA), and Morgan Stanley (MSFT shares). Each ingested differently and merged into a unified portfolio view.

🤖 LLM-Powered Research

Forward a tweet to Telegram → Claude extracts tickers → Opus runs deep research using SEC EDGAR, yfinance, and NewsAPI → thesis stored.

📱 Telegram Bot

Commands for portfolio, thesis, targets, and rebalance. Scheduled morning briefing at 9am ET and EOD digest at 4:30pm ET — all running locally on your Mac.

💾 SQLite (Local-Only)

All data stored on-device. No hosting, no tunneling, no external exposure. FastAPI runs on 127.0.0.1:8000. Bot calls the API over localhost.

⚡ Overlap Handling

Same ticker in Robinhood + Fidelity → total_shares = RH + Fidelity. Portfolio shows both combined and per-account views. Rebalance engine recommends selling from Fidelity first (tax-free) before touching Robinhood lots.

🏆 Investment Routing Preference

Rebalance suggestions always show which account the trade should happen in, and why.

1
Fidelity first (Roth IRA) — preferred for new buys; no tax drag; trade freely
2
Robinhood fallback — only when liquidity needed; margin available; STCG impact flagged on every suggested sell

🔄 Key Data Flows

1
Copy Robinhood portfolio text
pbpaste or manual paste → python scripts/ingest_rh.py --paste "$(pbpaste)"
2
Parse text with regex + dateparser
Extract tickers, shares, cost basis, current price. Auto-detect snapshot date from content.
3
SHA-256 content hash → deduplication check
If UNIQUE(account_id, content_hash) already exists → skip entirely. Safe to run twice.
4
INSERT raw_snapshots (immutable)
Raw paste text stored verbatim. Never overwritten. Serves as source of truth for re-parsing.
5
INSERT positions (one row per ticker)
Each position linked to its snapshot_id and account_id.
6
Enrich new tickers via yfinance
Fetch GICS sector, industry, live price. Cache in price_history. Update tickers table.
7
Recompute today's portfolio_snapshot
Aggregate all accounts → combined total value row. Used for time-series P&L charts.
1
Export from Fidelity (XLSX or CSV)
python scripts/ingest_fidelity.py --file ~/Downloads/Portfolio_Export.xlsx — auto-detects format: pandas.read_excel for .xlsx, pandas.read_csv for .csv
2
Fidelity column mapping (handles both export formats)
Maps Fidelity's column names to internal schema. Positions tagged account_id='fidelity' (Roth IRA — tax_free, illiquid).
3
Split: cash rows → cash_positions, equity rows → positions
Fidelity exports include cash as a "position." These are routed separately.
4
Same pipeline as Robinhood from step 3 onward
Dedup → raw_snapshots → positions → enrich → portfolio_snapshot.
1
You forward a tweet to the Telegram bot
Any free text or forwarded tweet is handled by bot/messages.py.
2
Tweet archived as timestamped Markdown file
Raw tweet text saved verbatim to data/raw/tweets/YYYYMMDD_HHMMSS.md before any processing.
3
POST /research → extractor.py (Claude Sonnet) — structured extraction
Extracts a full payload: tickers: [NVDA, TSLA] · intent: "bullish on AI capex cycle" · thesis: "NVDA benefits from..." (null if absent) · context: "macro/sector backdrop"
4
INSERT tweet_ingests row
Stores raw_text, file_path, tickers[], intent, thesis, and context. Immutable record of what the tweet said.
5
INSERT research_jobs (status='pending') for each new ticker
Jobs queued in DB. Background FastAPI task picks them up.
6
researcher.py (Claude Opus) — deep research per ticker
Pulls SEC EDGAR filings, yfinance fundamentals, NewsAPI headlines. Prompt caching on large context blocks. validated=false, raw output stored verbatim.
7
validators.py cross-checks LLM sector vs yfinance
If LLM claims NVDA is in "Healthcare" but yfinance says "Technology" → flag mismatch.
8
theme_accumulator.py (Claude Sonnet) — thematic clustering
Pulls all tweet_ingests + research_results for related tickers. Identifies or updates thematic clusters (e.g. "AI Capex Cycle", "Rate Sensitivity"). UPSERT themes rows with updated ground_truth summary.
9
Bot notifies you: tickers + intent summary + thesis preview
Formatted in Telegram MarkdownV2. Includes intent found and thesis preview if one was extracted.
1
You send: /thesis NVDA "add Q1 2025 earnings beat"
Handled by bot/commands.py → POST /thesis/NVDA
2
thesis/store.py fetches current version
Retrieves the latest thesis markdown for NVDA from thesis_documents.
3
thesis/editor.py (Claude Sonnet) with prompt caching
Current thesis text passed with cache_control: ephemeral → up to 90% cost reduction on follow-up edits.
4
content_hash dedup check
If new text is identical to current version → skip write. Prevents empty version bumps.
5
INSERT thesis_documents version N+1
Monotonically increasing version. Old versions never deleted. FTS5 index updated.
6
Bot replies: "Thesis updated (v4): [500 char preview]"
You see a preview inline in Telegram without leaving the app.
1
APScheduler fires at 4:30 PM ET
Also fires morning briefing at 9:00 AM ET. Price target poll every 5 minutes.
2
GET /portfolio → combined portfolio state
Merges RH + Fidelity + MS positions. Overlap tickers summed across accounts.
3
GET /portfolio/performance → day P&L per ticker
Compares today's prices vs prior close from price_history.
4
GET /targets → near-target alerts
Checks proximity to all active buy/sell/alert targets.
5

Portfolio Summary · Top Movers · Near Targets · LTCG Tax Reminders (RH positions crossing 1yr this week)

📐 Design Principles

1
Immutable Raw Data
raw_snapshots is append-only. Re-parsing from original content is always possible. Nothing is ever overwritten.
2
Event-Sourced Portfolio State
portfolio_snapshots is a materialized view recomputable from positions + price_history. Lose it? Rebuild it.
3
Idempotent Ingestion
UNIQUE(account_id, content_hash) enforced at DB level. Running ingestion twice is always safe — no duplicates, no errors.
4
LLM as Enrichment Layer
All LLM outputs stored verbatim with validated=false. LLM never owns authoritative fields until cross-validated by structured sources.
5
Tax-Aware Routing Everywhere
tax_treatment is first-class on accounts. Rebalance rules read it before every suggestion. STCG lots on Robinhood flagged explicitly.
6
Separation of Concerns
Bot only calls API. API only calls modules. Modules have no knowledge of bot or API. Each layer is independently testable.
7
Prompt Caching
Long context (SEC filings, thesis text) uses cache_control: ephemeral in Anthropic API messages — up to 90% cost reduction on repeated runs within the 5-min cache window.
